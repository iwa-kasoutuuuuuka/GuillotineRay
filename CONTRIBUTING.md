# Contributing to Guillotine Ray

Thank you for your interest in contributing to Guillotine Ray! We welcome all contributions that help make this tool more effective and stable.

## How to Contribute

1. **Bug Reports**: Open an issue describing the bug and steps to reproduce it.
2. **Feature Requests**: Open an issue explaining the feature and why it would be useful.
3. **Pull Requests**:
   - Fork the repository.
   - Create a new branch for your changes.
   - Ensure your code follows the existing style and memory management rules (always dispose `Mat` objects).
   - Submit a pull request.

## Development Rules
- Use .NET 8.
- Respect DPI awareness.
- Avoid platform-specific code outside of Windows (though this is currently a WinForms project).
- Keep performance in mind (optimize matching loops).
