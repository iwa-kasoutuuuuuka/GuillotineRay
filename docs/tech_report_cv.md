# 画像認識技術レポート：テンプレートマッチングの理論と実践

## 1. テンプレートマッチングの基礎
テンプレートマッチングとは、ソース画像（検索対象）の中から、テンプレート画像（探したい対象）と最も似ている箇所を特定する画像処理手法です。

本ツールでは、OpenCV の `MatchTemplate` 関数を利用し、スライディングウィンドウ方式で全画素を走査して類似度スコアを算出しています。

## 2. 正規化相関係数 (CCoeffNormed) の採用理由
OpenCV には複数のマッチング方式がありますが、本ツールでは **`CCoeffNormed` (正規化相互相関計数法)** を採用しています。

### 数学的特徴
この方式は、以下の数式に基づきスコアを `-1.0` から `1.0` の範囲で算出します。
- **正規化 (Normalized)**: 明るさの変動の影響を排除します。ソース画像の一部が少し暗くても、パターンの特徴が一致していれば高いスコアが出ます。
- **相関係数 (Coeff)**: 平均値を差し引いてから計算するため、テンプレートとソースの「形状の似具合」をより正確に捉えます。

## 3. 高速化の三原則
デスクトップのリアルタイム監視において、毎秒数十回の認識を行うには最適化が不可欠です。本プロジェクトでは以下の 3 点を適用しています。

### ① ROI (Region of Interest) 制限
画面全体（4K モニター等）を走査すると、計算量は数百万画素に及びますが、監視範囲を ROI (例: 300x300) に限定することで、スキャン範囲を **95% 以上削減** しています。

### ② グレースケール変換 (Dimensionality Reduction)
RGB 3チャンネルの情報は、人間の目には鮮やかですが、パターンマッチングにおいては計算量を 3 倍にする要因です。
- **理由**: 明るさのパターンさえあれば画像は識別可能です。グレースケール化により、1ピクセルあたりの計算量を 3分の1 に圧縮しつつ、認識精度を維持しています。

### ③ 前回のヒット位置キャッシュ
ゲームや操作ループにおいて、次に現れる対象は「前回と同じ画像」である確率が高いという経験則に基づき、前回ヒットしたテンプレートをループの最初にスキャンします。これにより、多種類の画像を登録していても、ヒット時の応答速度が劇的に向上します。

## 4. 限界と克服
テンプレートマッチングには以下の弱点があります。
- **回転・拡大縮小に弱い**: 数ピクセルでもサイズが変わるとスコアが急落します。
- **対策**: 本ツールでは「Snipping Tool」でピクセル完全一致の画像を作成することを前提としていますが、より柔軟な認識が必要な場合は「特徴点マッチング (SIFT/ORB)」や「AI/CNN (YOLO等)」への拡張が必要です。
